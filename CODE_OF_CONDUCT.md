# HighClaw Code of Conduct

## Our Commitment

HighClaw is committed to providing a welcoming, inclusive, and harassment-free environment for all contributors, regardless of experience level, background, identity, or personal characteristics.

We value technical excellence, constructive collaboration, and respectful communication.

## Expected Behavior

- **Be respectful.** Treat all participants with courtesy and professionalism.
- **Be constructive.** Provide helpful feedback; focus on improving code and ideas, not criticizing people.
- **Be collaborative.** Work together toward shared goals; assume good intent from others.
- **Be inclusive.** Welcome newcomers; help others learn and grow.
- **Be accountable.** Own your mistakes; correct them promptly and gracefully.

## Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment, intimidation, or discrimination of any kind
- Personal attacks, insults, or derogatory comments
- Trolling, disruptive behavior, or bad-faith arguments
- Publishing private information without consent
- Any conduct that would be considered inappropriate in a professional setting

## Scope

This Code of Conduct applies to all HighClaw project spaces, including:

- GitHub issues, pull requests, and discussions
- Project communication channels (Discord, Slack, etc.)
- Official project events and meetups
- Any context where you represent the project

## Reporting

If you experience or witness unacceptable behavior, please report it through:

- **GitHub:** Open a private security advisory or contact maintainers directly
- **Email:** Contact maintainers via the channels listed in the repository profile

All reports will be reviewed promptly and handled confidentially.

## Enforcement

Maintainers are responsible for enforcing this Code of Conduct. Actions may include:

1. **Warning** — A private notice explaining the violation
2. **Temporary restriction** — Limited participation in project spaces
3. **Permanent ban** — Removal from all project spaces

Enforcement decisions are made at the discretion of maintainers and are final.
